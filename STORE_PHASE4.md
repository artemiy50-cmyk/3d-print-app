# Store Landing — Фаза 4: Каталог, корзина, оформление заказа

План действий по реализации публичного каталога товаров в Store, корзины и оформления заказа.

---

## 1. Цели фазы

1. **Каталог** — отобразить товары из `storeProducts` на странице Store (сейчас блок пустой).
2. **Корзина** — добавить товары в корзину, изменить количество, удалить, показать итог.
3. **Оформление заказа** — форма контактов покупателя, отправка заказа в Firebase, подтверждение.

**Не входит в Фазу 4:** уведомления (Email, Telegram) — Фаза 6.

---

## 2. Решения по уточняющим вопросам

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Корзина | Иконка в шапке + выдвижная панель (drawer) **и** отдельная страница `/cart` |
| 2 | Где продавец видит заказы | В «Мой магазин» — справа от настроек (просторная область) |
| 3 | Изображения | Отдельные, не связаны с `data/products`. В `storeProducts` — поле `imageUrl` (опционально) |
| 4 | Минимальная сумма заказа | Настройка магазина: `minOrderAmount` (опционально, ₽) |
| 5 | Подтверждение покупателю | Только «Заказ отправлен» на экране. Email — в Фазе 5 |

---

## 3. Модель данных

### 3.1 Текущие данные (уже есть)

- `storeProductsData` — массив `{ name, price, productId?, inCatalog, imageUrl? }` (загружается в store.js). Изображения отдельные, не связаны с `data/products`.
- `storeConfig` — настройки магазина (title, description, contactEmail, contactTelegram, **minOrderAmount?** — минимальная сумма заказа, ₽).

### 3.2 Новое: заказы

**Путь:** `storeOrders/{orderId}` (корень Realtime Database)

**Структура заказа:**
```json
{
  "ownerUid": "abc123...",
  "subdomain": "test-shop",
  "items": [
    { "name": "Товар 1", "price": 500, "qty": 2, "productId": 12345 }
  ],
  "total": 1000,
  "buyerName": "Иван",
  "buyerEmail": "buyer@example.com",
  "buyerPhone": "+79001234567",
  "comment": "Доставка до двери",
  "createdAt": "2026-02-28T12:00:00.000Z",
  "status": "new"
}
```

**Правила Firebase:**
- `.read` — только владелец (`auth.uid === data.child('ownerUid').val()`).
- `.write` — создание без авторизации (покупатель анонимно отправляет заказ).

## 4. План работы

### 4.1 Каталог

- Разметка внутри `#storeCatalog`: сетка карточек товаров.
- Карточка: наименование, цена (₽), кнопка «Добавить в корзину». Изображение — из `imageUrl` или заглушка.
- При пустом каталоге — текст «В каталоге пока нет товаров».

### 4.2 Корзина

- Иконка корзины в шапке Store — показывает количество товаров (и/или итог).
- При клике — **выдвижная панель** (drawer): список товаров, количество (+/−), удаление, итог, кнопка «Оформить заказ».
- **Отдельная страница `/cart`** — полный вид корзины с той же логикой.
- Данные корзины — в `localStorage` или в памяти (сессия).
- Проверка `minOrderAmount` — при оформлении, если настроено.

### 4.3 Оформление заказа

- Форма: Имя*, Email*, Телефон (необ.), Комментарий.
- Кнопка «Отправить заказ» → запись в `storeOrders` через `push()` → сообщение «Заказ отправлен», очистка корзины.
- Продавец видит заказы в Manager: **«Мой магазин» — справа от настроек** (область заказов).

### 4.4 Файлы

- `store.html` — разметка каталога, выдвижная панель корзины, страница `/cart`, форма заказа.
- `store.js` — роутинг (в т.ч. `/cart`), рендер каталога, логика корзины, отправка заказа.
- `index.html`, `script.js` — блок заказов в «Мой магазин» (справа от настроек).
- `style.css` — стили `.store-card`, `.store-cart`, `.store-cart-drawer`, `.store-checkout`.
- `database.rules.json` — правила для `storeOrders`.

## 6. Критерии приёмки

- [x] На странице Store отображается каталог товаров (наименование, цена, кнопка «Добавить в корзину»).
- [x] При пустом каталоге показывается сообщение «В каталоге пока нет товаров».
- [x] Иконка корзины в шапке + выдвижная панель (drawer) + отдельная страница `/cart` (#cart).
- [x] Корзина: добавление товара, изменение количества, удаление, отображение итога.
- [x] Настройка `minOrderAmount` в магазине; проверка при оформлении.
- [x] Оформление заказа: форма (имя, email, телефон, комментарий), отправка в Firebase.
- [x] После отправки заказа — сообщение «Заказ отправлен», корзина очищается.
- [x] Заказы сохраняются в `storeOrders` с `ownerUid`, `subdomain`, `items`, `total`, контакты, `createdAt`.
- [x] Продавец видит заказы в «Мой магазин» справа от настроек магазина.
