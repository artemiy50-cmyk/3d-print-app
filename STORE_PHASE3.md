# Store Landing — Фаза 3: Раздел «Мой магазин» в Manager

План действий по добавлению раздела «Мой магазин» в Manager — настройка магазина, резервирование поддомена, управление товарами в каталоге.

---

## 1. Цели фазы

1. Добавить пункт меню **«Мой магазин»** в sidebar Manager.
2. Реализовать страницу настройки магазина: поддомен, заголовок, описание, контакты, включение/выключение.
3. Реализовать регистрацию поддомена (запись в `storesBySubdomain`).
4. Реализовать управление товарами в каталоге: добавление изделий из `products` в `storeProducts` с ценой, удаление.

**Не входит в Фазу 3:** рендер каталога в Store (Фаза 4), корзина и оформление заказа.

---

## 2. Модель данных (напоминание)

| Путь | Назначение |
|------|------------|
| `storesBySubdomain/{subdomain}` | индекс: поддомен → ownerUid |
| `users/{uid}/store` | настройки магазина |
| `users/{uid}/storeProducts` | товары в каталоге (productId, price, inCatalog) |
| `users/{uid}/data/products` | изделия пользователя (источник для добавления) |

---

## 3. План работы

### 3.1 Изменения в index.html и script.js

- Добавить пункт меню «Мой магазин» в sidebar — **ниже иконки личного кабинета (👤)** в `setupUserSidebar()`, т.е. после profile, перед Выйти.
- Добавить страницу (div.page) с id `myStore`.

### 3.2 Контент страницы «Мой магазин»

**Блок А: Настройки магазина**

- **Поддомен** — поле ввода (если магазин не создан) или только отображение (если уже создан). **После создания смена запрещена.** Подсказка: «Для смены поддомена обратитесь к администратору» — ссылка «Связаться» есть в меню.
- **Заголовок** — текст.
- **Описание** — текст (textarea).
- **Магазин включён** — переключатель (checkbox/switch).
- **Email для связи** — необязательно.
- **Telegram для связи** — необязательно.
- Кнопка **«Сохранить»**.

**Блок Б: Товары в каталоге**

- Таблица: изделие, цена, действия (удалить).
- Кнопка **«Добавить товар»** → выбор изделия из `products` (в наличии), ввод цены, добавление в `storeProducts`.
- Ссылка на магазин: `test-shop.my-3d-print.ru` (или `store.html?store=test-shop` для localhost).

### 3.3 Логика в script.js

- `fillStoreSettingsPage()` — загрузка `users/{uid}/store` и `storeProducts`, рендер формы.
- `saveStoreSettings()` — сохранение в `users/{uid}/store`, при первом создании — запись в `storesBySubdomain`.
- `addStoreProduct(productId, price)` — добавление в `storeProducts`.
- `removeStoreProduct(index)` — удаление из `storeProducts`.
- Валидация поддомена: только a-z, 0-9, дефис; не из зарезервированных (см. 3.5); проверка занятости в `storesBySubdomain`.

### 3.4 Смена поддомена вручную (через администратора)

При необходимости смены поддомена администратор выполняет в Firebase Console (Realtime Database):

1. Удалить `storesBySubdomain/{старый_поддомен}`
2. Добавить `storesBySubdomain/{новый_поддомен}` → `{ ownerUid: "<uid>" }`
3. Обновить `users/{uid}/store/subdomain` → `"новый_поддомен"`

Пользователю сообщить: «Для смены поддомена обратитесь к администратору через ссылку «Связаться» в меню».

## 6. Критерии приёмки

- [ ] В sidebar Manager есть пункт «Мой магазин», при клике открывается соответствующая страница.
- [ ] Если магазин уже создан (например, через seed Фазы 2): отображаются текущие настройки (поддомен, заголовок, описание, enabled, контакты), их можно редактировать и сохранять.
- [ ] Если магазин не создан: можно ввести поддомен, заголовок, описание, нажать «Сохранить» — создаются записи в `storesBySubdomain` и `users/{uid}/store`.
- [ ] Переключатель «Магазин включён» сохраняется в `store.enabled`; при `enabled === false` Store показывает «Магазин не найден» (уже работает в Фазе 2).
- [ ] Можно добавить товар: выбор изделия из `products` (в наличии), ввод цены — запись в `storeProducts`.
- [ ] Можно удалить товар из каталога — удаление из `storeProducts`.
- [ ] Отображается ссылка на магазин (или подсказка, как открыть Store по поддомену).
- [ ] Поддомен валидируется: только a-z, 0-9, дефис; не из зарезервированных; при занятости другим пользователем — сообщение об ошибке.
